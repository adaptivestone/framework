# P1x — v5.2.1 adoption fixes

**Status**: 🟢 implemented 2026-07-20 · complete release gate green · awaiting `5.2.1`
**Target**: `5.2.1`, additive/fix
**Depends on**: shipped 5.2.0 route groups and OpenAPI schema resilience

## Goal

Close two integration gaps found while adopting the 5.2.0 features: grouped controller files must
retain framework override identity, and first-party Pagination parameters must appear in OpenAPI.
The related literal route-getter compatibility fix is tracked separately as P1w.

## Delivered behavior

- Controller override comparison ignores fully parenthesized route-group segments at runtime and
  during AST discovery; returned physical paths and generated-file placement remain unchanged.
- The generic inheritance loader accepts an optional comparison-only normalizer, so models,
  configs, commands, and migrations keep exact relative-path semantics.
- `defineSchema(validate, { jsonSchema })` exposes an optional JSON Schema object or factory through
  the existing Standard Schema driver without adding a validator dependency.
- Pagination uses that seam to emit optional numeric `page` and `limit` query parameters without
  introspection warnings; runtime parsing, fallback, and clamping are unchanged.

## Files

- `src/helpers/files.ts`, `src/modules/Base.ts`, `src/modules/AbstractController.ts` — scoped
  override normalization.
- `src/controllers/index.ts`, `src/codegen/astEmit.ts` + tests — runtime/AST grouped overrides.
- `src/services/validate/defineSchema.ts`, `StandardSchemaDriver.test.ts` — explicit JSON Schema.
- `src/services/http/middleware/Pagination.ts`, `OpenApiGenerator.test.ts` — first-party parameters.
- Canonical documentation, example application, changelog, and release handoff.

## Out of scope

- Treating route groups as namespaces or changing explicit controller mount paths.
- Applying controller grouping semantics to any non-controller inheritance loader.
- Inferring JSON Schema from imperative validator code.
- Changing Pagination's accepted values or effective paging behavior.

## Verification
